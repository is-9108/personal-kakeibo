## 1. システム概要・アーキテクチャ

### 1.1 システム概要

個人利用を前提とした家計簿Webアプリケーション。毎日の収支入力を継続できるシンプルなUXを最優先とし、固定費の自動登録・月次レポートの自動生成により手間を最小化する。

- 利用者：シングルユーザー（本人のみ）
- 想定デバイス：スマートフォン（主）・PC（副）
- 稼働環境：AWS Lightsail（常時起動、月額 $3.5〜$7）

### 1.2 技術スタック

| レイヤー | 技術 | バージョン目安 |
|---------|------|--------------|
| フロントエンド | Streamlit | 1.x |
| バックエンド | FastAPI (Python) | 0.110.x |
| データベース | SQLite | 3.x |
| グラフ描画 | Plotly | 5.x |
| デプロイ | AWS Lightsail | - |
| スケジューラ | APScheduler（FastAPIに組み込み） | 3.x |
| ジョブストア | SQLAlchemyJobStore（SQLite）| - |

## 2. 機能要件

### 2.1 ユーザーストーリー一覧

| ID | ユーザーストーリー |
|----|-----------------|
| US-01 | 私は、毎日の収支を素早く入力したい。なぜなら、入力に時間がかかると継続できなくなるから。 |
| US-02 | 私は、誤って入力した収支を修正・削除したい。なぜなら、記録の正確性を保つ必要があるから。 |
| US-03 | 私は、カテゴリ別の支出割合を円グラフで確認したい。なぜなら、どこにお金を使っているかを視覚的に把握したいから。 |
| US-04 | 私は、家賃とNISAを毎月自動で登録したい。なぜなら、毎月必ず発生する固定費を手動入力する手間を省きたいから。 |
| US-05 | 私は、固定費の金額・登録日を設定画面で変更したい。なぜなら、金額が変わったときに対応できるようにしたいから。 |
| US-06 | 私は、毎月25日に自動生成された月次レポートを確認したい。なぜなら、月ごとの収支トレンドを把握したいから。 |
| US-07 | 私は、先月比の収支変化を確認したい。なぜなら、支出が増減した原因を振り返りたいから。 |

### 2.2 機能一覧

#### 2.2.1 収支入力機能（US-01）

- 入力フォームに以下の項目を表示する
  - 日付（デフォルト：本日）
  - 収支区分（収入 / 支出）
  - カテゴリ（収支区分に連動してプルダウン切替）
  - 金額（円単位、1以上の整数）
  - 支払い方法（楽天Pay / クレジットカード / 現金）
  - メモ（任意、最大200文字）
- 登録ボタン押下で即時保存し、フォームをリセットする
- 入力バリデーション：金額は1円以上の整数のみ許可

#### 2.2.2 収支編集・削除機能（US-02）

- 直近30件の収支一覧を表形式で表示する
- 一覧の各行に「編集」「削除」ボタンを設ける
- 編集：既存の値を初期値として入力フォームを表示し、保存で上書きする
- 削除：確認ダイアログを表示してから削除する

#### 2.2.3 カテゴリ別円グラフ表示（US-03）

- 表示対象期間を「今月」「先月」「任意月指定（年月選択）」から選択できる
- 支出のカテゴリ別合計金額をPlotlyの円グラフで表示する
- グラフ下部に各カテゴリの金額・割合を表形式で表示する

#### 2.2.4 固定費自動登録機能（US-04）

- APSchedulerのcronジョブとして実装する
- 固定費設定テーブルに登録された各レコードを毎月指定日の09:00 JSTに収支トランザクションとして自動登録する
- 重複登録を防ぐため、同月・同固定費IDのレコードが既に存在する場合はスキップする
- **APScheduler設定の注意点：**
  - ジョブストアに `SQLAlchemyJobStore` を使用し、FastAPI再起動時の二重登録を防ぐ
  - ジョブ追加時は `replace_existing=True` を指定する
  - タイムゾーンは `timezone='Asia/Tokyo'` を明示的に設定する

#### 2.2.5 固定費設定機能（US-05）

- 固定費（家賃・NISA）の以下の項目を設定画面で変更できる
  - 金額（円単位）
  - 毎月の登録日（1〜28日の整数）
  - 有効/無効フラグ
- 初期データとして「家賃」「NISA」の2件をシードデータとして投入する

#### 2.2.6 月次レポート自動生成・閲覧機能（US-06・US-07）

- APSchedulerのcronジョブとして毎月25日 09:00 JSTに実行する
- **生成対象月：毎月25日に「前月分」のレポートを生成する**（例：3月25日 → 2月分のレポートを生成）
- 生成内容
  - 対象月の収入合計・支出合計・収支差額
  - カテゴリ別支出合計（金額・割合）
  - 先月比：収入・支出・各カテゴリ別の増減額と増減率
- **初回レポート（前月データが存在しない場合）の先月比処理：**
  - `prev_month_diff` は `null` を返す
  - 画面上は「先月比データなし（初月のため）」と表示してスキップする
- レポートはDBのreportsテーブルに保存する
- アプリ内の専用ページで過去レポートを月単位で選択・閲覧できる
- 閲覧ページにカテゴリ別円グラフ（Plotly）と先月比テーブルを表示する
- 対象月の25日以前に手動生成ボタンで即時生成することもできる

## 3. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンシブデザイン | Streamlitのレイアウト機能を用いてスマートフォン（幅360px以上）・PC（幅1024px以上）の両方で操作可能にする |
| パフォーマンス | 収支一覧・グラフの初期表示を3秒以内に完了する（Lightsail最小構成相当） |
| 可用性 | Lightsail上でプロセスが停止した場合、systemdによる自動再起動により5分以内に復旧する |
| データ保全 | SQLiteのデータベースファイルを毎日1回cronで自動バックアップする。コマンド：`0 3 * * * sqlite3 /path/to/kakeibo.db ".backup /path/to/backup/kakeibo_$(date +\%Y\%m\%d).db"` を crontab に登録。バックアップファイルは直近7世代を保持し、古いものは自動削除する。 |
| セキュリティ | 個人利用のため認証機能は不要。Lightsailのファイアウォール設定でStreamlitポート（8501）へのアクセスを自分のIPアドレスに限定する。FastAPIポート（8000）はlocalhost（127.0.0.1）からのみアクセス可能とし、外部に公開しない。StreamlitはFastAPIに対して `http://localhost:8000` で内部通信する。 |
| UX | 毎日の入力フォームをトップページに配置し、3タップ以内で1件の収支を登録できる構成にする |
| 拡張性 | カテゴリはDBで管理し、コード修正なしにDBレコード追加でカテゴリを増やせる設計にする |

## 5. API仕様

ベースURL：`http://localhost:8000/api/v1`

### 5.1 収支トランザクション

| メソッド | エンドポイント | 説明 |
|---------|-------------|------|
| GET | `/transactions` | 収支一覧取得 |
| POST | `/transactions` | 収支新規登録 |
| PUT | `/transactions/{id}` | 収支更新 |
| DELETE | `/transactions/{id}` | 収支削除 |

#### GET /transactions

クエリパラメータ：

| パラメータ | 型 | 必須 | 説明 |
|-----------|---|------|------|
| year | integer | NO | 絞込：年（例：2026） |
| month | integer | NO | 絞込：月（1〜12） |
| limit | integer | NO | 取得件数（デフォルト：30、最大：365） |
| offset | integer | NO | オフセット（デフォルト：0） |

レスポンス例（200 OK）：

```json
{
  "total": 5,
  "items": [
    {
      "id": 1,
      "date": "2026-03-20",
      "type": "expense",
      "category_id": 1,
      "category_name": "食費",
      "amount": 980,
      "payment_method_id": 1,
      "payment_method_name": "楽天Pay",
      "memo": "コンビニ",
      "fixed_cost_id": null,
      "created_at": "2026-03-20T12:00:00+09:00",
      "updated_at": "2026-03-20T12:00:00+09:00"
    }
  ]
}
```

#### POST /transactions

リクエストボディ：

```json
{
  "date": "2026-03-20",
  "type": "expense",
  "category_id": 1,
  "amount": 980,
  "payment_method_id": 1,
  "memo": "コンビニ"
}
```

バリデーション：
- `date`：YYYY-MM-DD形式の有効な日付
- `type`：`income` または `expense` のいずれか
- `amount`：1以上の整数
- `memo`：200文字以内（任意）

レスポンス（201 Created）：登録したトランザクションオブジェクトを返す

#### PUT /transactions/{id}

リクエストボディ：POST と同一スキーマ

レスポンス（200 OK）：更新後のトランザクションオブジェクトを返す

エラー（404 Not Found）：指定IDが存在しない場合

#### DELETE /transactions/{id}

レスポンス（204 No Content）

エラー（404 Not Found）：指定IDが存在しない場合

```json
  "generated_at": "2026-02-25T09:00:00+09:00"
}
```

エラー（404 Not Found）：指定月のレポートが存在しない場合

#### POST /reports/{year}/{month}/generate

レスポンス（201 Created）：生成・更新されたレポートオブジェクトを返す
